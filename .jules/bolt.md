## 2024-03-24 - ResultCache dictionary iteration optimization
**Learning:** In the `ResultCache._create_new_block` method of `tools/gui/gui_helpers.py`, eviction involves iterating over all keys in the oldest block. Previously, it used `tuple(old["entries"].keys())` which allocates a new tuple object on every eviction cycle. Because `old["entries"]` is only read and not structurally modified during the loop (the inner logic pops keys from `_key_index`, not `old["entries"]`), the tuple allocation is an unnecessary O(N) memory allocation and traversal in a performance-critical loop.
**Action:** When iterating over a dictionary merely to read its keys without modifying the dictionary itself, iterate directly over the dictionary (e.g., `for key in entries:`) instead of casting its keys to a tuple or list, saving allocations and CPU cycles.
