## 2024-04-10 - O(N) Quartile Extraction in PlotEngine
**Learning:** `std::sort` was used to fully sort a fixed-size `FixedVector<double, 256>` array simply to extract quartiles for BoxPlots. Since box plots only need the min, max, median, and 25th/75th percentiles, a full $O(N \log N)$ sort is unnecessary. `std::nth_element` combined with `std::minmax_element` can achieve the same result in $O(N)$ average time. However, when partitioning sub-ranges (e.g., for Q1 and Q3) with `std::nth_element`, explicit boundary checks (`if (q1_idx < med_idx)`) are crucial to prevent undefined behavior and crashes on small arrays where indices might overlap or fall out of bounds.
**Action:** Use `std::nth_element` for order statistics instead of full sorts, but always add boundary guards when computing percentiles dynamically to ensure valid iterator ranges.
