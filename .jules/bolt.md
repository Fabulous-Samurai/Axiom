## 2024-04-09 - Use O(N) std::nth_element instead of O(N log N) std::sort for statistical computations
**Learning:** In `StatisticsEngine` and `PlotEngine`, calculating quantiles (Median, Percentile, Box Plot data) was originally implemented by fully sorting the input arrays, taking O(N log N) time. For just extracting specific partition values, `std::nth_element` guarantees the same output with average O(N) time complexity without sacrificing code readability. When extracting quartiles with multiple `std::nth_element` calls, passing relative sub-array iterators can accidentally cause out-of-bounds UB if bounds are not strictly respected; it is generally safer to run it over the entire bounds `[begin, end]` since its O(N) nature guarantees minimal overhead anyway.
**Action:** When extracting min, max, median, and arbitrary percentiles, actively scan the codebase for `std::sort` calls. Replace them with combination of `std::minmax_element` and `std::nth_element`. Always verify bounds checking for iterator offsets when running multiple `std::nth_element` passes.
