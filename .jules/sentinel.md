## 2024-03-24 - Sandbox Arbitrary Code Execution
**Vulnerability:** The sandbox evaluator in `scripts/sandbox.py` previously executed user expressions by evaluating them directly via Python's built-in `eval()` in an isolated subprocess. This allowed arbitrary code execution, enabling an attacker to import modules (like `os`) and access files/systems simply by bypassing the subprocess barrier using Python commands.
**Learning:** Using `eval` is extremely dangerous even when separated by a process boundary. Direct string evaluations will evaluate any valid Python statement, circumventing process isolation if a Python interpreter invokes it without proper system-level constraints (like seccomp/AppContainer).
**Prevention:** Always parse expressions to an Abstract Syntax Tree (AST) using `ast.parse` to whitelist safe operations (like math functions) and completely reject unsupported node types (like `ast.Call` or `ast.Attribute`). Additionally, remember to set `sys.set_int_max_str_digits(0)` before computationally heavy evaluations to avoid Python string conversion limits.
