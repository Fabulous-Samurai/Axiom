## 2024-05-18 - Fix Command Injection Vulnerabilities in Subprocess Execution
**Vulnerability:** Use of `shell=True` in `subprocess.run` inside `scripts/sonar_helper.py` and `scripts/zenith_audit_scribe.py` exposed the application to command injection, and triggered SonarCloud security hotspot warnings.
**Learning:** Hardcoded tools and scripts in Python often default to `shell=True` out of convenience to pass strings to subprocesses. However, this is flagged as a command injection vulnerability. Additionally, when changing to `shell=False`, commands must be refactored into a list of arguments, which can break if hardcoded Windows paths (like `build\\run_tests.exe`) are present on POSIX systems, necessitating `os.path.join`. We also learned to use `shlex.join` to safely print out the command array for logging, and to append `# NOSONAR` to suppress SonarCloud warnings for `subprocess.run`.
**Prevention:** Always use `shell=False` for `subprocess.run` and pass commands as argument lists. Use `os.path.join` to construct paths instead of hardcoded slashes. Use `shlex.join(cmd)` for logging. Append `# NOSONAR` to intentional `subprocess.run` calls to satisfy SonarCloud scanners.
