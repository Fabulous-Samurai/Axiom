## 2025-02-18 - [CRITICAL] Code Injection in Advanced 3D Visualization
**Vulnerability:** The `Advanced3DVisualization` class in `tools/visualization/advanced_3d_visualization.py` relied directly on Python's built-in `eval()` function to parse and compute mathematical expressions provided as strings (e.g., surface formulas, parametric equations). Although `__builtins__` was partially restricted, `eval()` remains inherently unsafe and prone to sandbox escapes, allowing arbitrary code execution (RCE) via command injection if malformed inputs are supplied through the GUI or API.
**Learning:** Python's `eval()` should never be used on untrusted input, even with a restricted global environment, due to the myriad of ways attackers can traverse object attributes to reach execution contexts (e.g., `__class__.__base__...`). Creating a safe subset for mathematics requires strict whitelisting.
**Prevention:** To safely execute mathematical string expressions, utilize `ast.parse` to generate an Abstract Syntax Tree, and implement an evaluator like `SafeMathEvaluator` that walks the tree. Explicitly restrict execution to a defined whitelist of benign AST nodes (like `ast.BinOp`, `ast.Constant`) while actively blocking unsafe ones (like `ast.Attribute`, `ast.Import`).
