## 2024-05-01 - Prevented Sandbox Escape/RCE in `scripts/sandbox.py`
**Vulnerability:** A Remote Code Execution (RCE) / Sandbox Escape existed in `scripts/sandbox.py` because Python's built-in `eval()` function was being used directly to evaluate arbitrary, potentially attacker-controlled mathematical expressions in a subprocess. An attacker could run `__import__('os').system('...')` to escape the environment.
**Learning:** `eval()` should never be used to evaluate untrusted input, even within isolated components or restricted subprocesses. While the subprocess isolates standard execution, the lack of input sanitization allowed complete command injection.
**Prevention:** Implement an Abstract Syntax Tree (AST)-based evaluator (`ast.parse`) combined with a strict node whitelist (like `SafeMathEvaluator`). Explicitly disallow `ast.Call` and `ast.Attribute` to prevent executing functions or accessing attributes, restricting the evaluation strictly to constants and basic operators.
