## 2025-03-08 - Prevent Sandbox Escape via `eval()` in Visualization Suite
**Vulnerability:** The advanced 3D visualization suite (`tools/visualization/advanced_3d_visualization.py`) was directly using `eval()` on user-supplied strings representing mathematical functions. By passing malicious functions like `__import__('os').system('...')`, a sandbox escape and arbitrary code execution was achievable. The `SafeMathEvaluator` existed in memory rules but was missing from this script.
**Learning:** Python's `eval()` is highly dangerous even with empty `__builtins__` if attribute access is allowed, as properties of loaded objects (like arrays or modules) can be exploited to walk up the object graph and gain execution privileges.
**Prevention:** Consistently apply `SafeMathEvaluator` for all mathematical expressions originating from dynamic/user input. This parser strictly filters allowable Abstract Syntax Tree (AST) nodes, specifically banning `ast.Attribute` and any node outside a whitelisted set of standard mathematical operations, effectively preventing object introspection and execution escapes.
