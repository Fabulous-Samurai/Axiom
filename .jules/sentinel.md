## 2025-02-28 - [CRITICAL] Fix code injection in Python tools
**Vulnerability:** Python scripts (like `scripts/sandbox.py`) executing mathematical expressions were using insecure `eval()` calls with dynamically constructed input strings (e.g., via `python -c "import os; print(eval(...))"`), creating a severe Remote Code Execution (RCE) vulnerability that could escape the intended sandbox.
**Learning:** Despite intended system-level sandboxing (like seccomp/AppContainer), the inner Python evaluation mechanism itself was insecure. The test suite correctly identified that operations like OS access were possible within the Python process if it wasn't externally restricted.
**Prevention:** Replaced `eval()` with an AST-based `SafeMathEvaluator` that parses expressions strictly in `mode='eval'` and explicitly allows only a predefined set of safe AST nodes (`ast.Constant`, `ast.BinOp`, etc.). The builtins dictionary was also explicitly cleared during evaluation (`{'__builtins__': {}}`) to provide an additional layer of defense against built-in function abuse.
