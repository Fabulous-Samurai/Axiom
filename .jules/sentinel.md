## 2025-05-03 - [CRITICAL] Prevent Remote Code Execution in Python Sandbox via `eval`
**Vulnerability:** A critical security vulnerability in `scripts/sandbox.py` allowed remote code execution (RCE) because user-provided expressions were directly evaluated using Python's `eval()` function inside a subprocess (`eval(repr(expression))`), making sandbox escapes like `__import__('os').listdir('.')` possible.
**Learning:** To successfully implement AST-based whitelist evaluation (`SafeMathEvaluator`) in constrained Python sandboxes, one must handle massive integer conversions that are commonly used as CPU-exhaustion (DoS) techniques. The mitigation requires `sys.set_int_max_str_digits(0)` before AST parsing. Although disabling limit mitigations is typically an anti-pattern, fixing a complete RCE takes priority, and AST constraints safely restrict parsing bounds.
**Prevention:** Never pass unfiltered user input to `eval()`. Always use an Abstract Syntax Tree (AST) parser (like `ast.parse` with `ast.NodeVisitor`) with an explicit node whitelist to evaluate mathematical or string expressions securely in python environments.
