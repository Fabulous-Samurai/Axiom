## 2024-05-01 - [RCE in Sandbox Evaluator]
**Vulnerability:** The sandbox executor (`scripts/sandbox.py`) used Python's native `eval()` function inside a subprocess to process arbitrary math expressions. While run in a separate process, this still allowed arbitrary python code execution, such as command execution via `__import__('os').system('...')`.
**Learning:** Python's `eval()` should never be used on untrusted inputs, even in sandbox environments. Instead, the `ast` module can be safely used to implement a restricted mathematical evaluator. Furthermore, when blocking standard library imports to prevent RCE, traditional timeout testing (like using `__import__('time').sleep(10)`) will fail. This requires switching to CPU-blocking calculations (e.g., `9**9999999`) paired with `sys.set_int_max_str_digits(0)` for proper timeout verification.
**Prevention:** Use an AST-based allowlist (`SafeMathEvaluator`) for mathematical evaluation. When writing tests for strict sandbox environments, utilize heavy mathematical evaluations to trigger computation timeouts rather than relying on system sleeps or function calls.
