## 2024-05-22 - [Fix arbitrary code execution via eval()]
**Vulnerability:** In `tools/visualization/advanced_3d_visualization.py`, mathematical expressions were being evaluated directly using Python's built-in `eval()`. Although `eval()` was called with `{"__builtins__": {}}` and a restrictive variable dictionary, malicious users could still execute arbitrary code and bypass the sandbox by utilizing attribute introspection (like `__class__` and `__bases__`) to access the `__import__` function or other dangerous system objects.
**Learning:** Limiting variables and clearing `__builtins__` in `eval()` is insufficient to prevent remote code execution in Python because objects inherently have attributes that allow escaping to the root object hierarchy.
**Prevention:** To safely evaluate arbitrary strings containing math functions, use `ast.parse(expression, mode='eval')` to parse the string into an Abstract Syntax Tree (AST), then strictly validate the nodes in the tree. Explicitly block `ast.Attribute` traversal to prevent any possibility of sandbox escape via introspection.
